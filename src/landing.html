The gap

AI agents are already acting on your behalf — deploying, triaging, messaging, updating records. They're doing it with borrowed credentials and implicit trust.

01

Shared secrets

Agents run on copied API keys and session cookies — credentials that grant everything the human can do, with no notion of "just this much."

02

No paper trail

When an agent acts, there's rarely a verifiable record of who authorized what, under which limits, or when that mandate expired.

03

No off switch

Pulling authority back means rotating keys across every system — slow, lossy, and impossible to do precisely for one agent.

The grant

A power of
attorney, written
for machines.

An APOA grant is a signed document where a principal delegates a narrow, time-bound mandate to an agent. Human-readable, machine-verifiable, and built from primitives you already trust.

grant.apoa.json ed25519 
{  "principal": "did:web:acme.com",  "agent":     "did:key:z6MktR…a91k",  "scope": ["deployments:create", "logs:read"],  "limits": {    "environment": "staging", "max_changes": 10,    "repositories": ["acme/api"], "production": false  },  "issued":  "2026-06-01T09:00:00Z",  "expires": "2026-07-01T00:00:00Z",  "revocable": true,  "revocation": "https://acme.com/apoa/revoke"}  // + detached ed25519 signature
The handshake

Four steps from intent to action.

A grant flows from principal to agent to verifier — and only then to the world. Every hop is checkable on its own.

Principal

Issue

You define a scope and sign a grant for a specific agent.

Agent

Present

The agent attaches the grant when it makes a request.

Verifier

Check

The service validates signature, scope, limits, expiry, revocation.

Ledger

Execute & log

If valid, the action runs — and is recorded against the grant.

Capabilities

What the standard guarantees.

01

Scoped permissions

Authority is a list of named actions with limits — never a blanket "act as me." Least privilege by construction.

02

Cryptographic proof

Grants are signed by the principal; every agent request is signed by the agent. No bearer-secret theft.

03

Full audit trail

Each action ties back to a grant ID, a scope entry, and a timestamp. Reconstruct exactly what happened.

04

Instant revocation

Hit the revocation endpoint and the grant dies everywhere that checks it — no key rotation, no blast radius.

05

Time & context bounds

Hard expiry plus quantitative and contextual caps — rate, environment, repository — enforced before the action, not after.

06

Built on open primitives

JSON, JWT, DIDs, ed25519 — a profile over standards you already run. No new trust root to adopt.

Why a new standard

OAuth was built for apps. Agents are different.

Existing auth assumes a human at the end of the flow and a long-lived relationship. Agents are ephemeral, numerous, and act unattended — they need delegation as a first-class object.

  API keys OAuth scopes APOA
Granularity All-or-nothing Coarse scopes Per-action + limits
Bounded actions — — ✓ Rate & context caps
Per-request proof Bearer secret Bearer token ✓ Signed by agent
Audit by mandate Logs only Logs only ✓ Tied to a grant
Instant revoke Rotate key Token revoke ✓ Per-grant, precise
Delegation chains — — ✓ Agent → sub-agent
In practice

Where a mandate matters.

Engineering

Release automation

Let a release agent deploy to staging, roll back failing builds, and never touch production.

deploy:stagingrollback:stagingno prod
Operations

Incident response

A diagnostic agent reads logs, opens incidents, and restarts approved workers without secrets.

logs:readincidents:createrestart:approved
Personal

Scheduling & inbox

An assistant books meetings and replies to known contacts, never sends to new recipients.

calendar:writemail:replyknown only
Governance

Access reviews

A review agent reads policy docs and opens change requests, but cannot merge or approve.

docs:readprs:createno merge
Support

Customer service

A support agent triages tickets, drafts replies, and updates status — no account deletions.

tickets:triagereply:draftno delete
Research

Data & tooling

A research agent queries read-only datasets and calls approved internal tools within a rate limit.

data:readtools:call100/hr
